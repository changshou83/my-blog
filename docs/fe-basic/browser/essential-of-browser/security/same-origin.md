- 浏览器安全可以分为三大块
	- Web 页面安全
	- 网络安全
	- 系统安全
- 同源策略（Same-origin policy）
	- 如果两个 URL 的协议、域名和端口都相同，我们就称这两个 URL 同源
	- 同源策略主要体现在DOM，Web数据和网络三层面
		- DOM：无法操作不同源的页面的DOM
		- Web数据：无法读取不同源的站点的Cookie，IndexDB，WebStorage等数据
		- 网络：无法将站点的数据发送给不同源的站点
- 安全和便利性的权衡，浏览器出让了同源策略的哪些安全性
	- 页面中可以嵌入第三方资源(Web数据)
		- 可能会被插入恶意代码进行XSS攻击
		- 为了解决 XSS 攻击，浏览器中引入了内容安全策略，称为 CSP
			- 核心思想是让服务器决定浏览器能够加载哪些资源，让服务器决定浏览器是否能够执行内联 JavaScript 代码
	- 跨域资源共享(网络)
		- 进行跨域访问控制，从而使跨域数据传输得以安全进行
	- 跨文档消息机制(DOM)
		- 可以通过 window.postMessage 的 JavaScript 接口来和不同源的 DOM 进行通信
