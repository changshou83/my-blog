课程内容

  • 错误处理是一项必要的功能
  • JS 通过错误对象来捕捉错误,通过throw语句抛出错误
    • throw 语句
      1. 暂停函数执行
      2. 将错误对象抛入错误栈
      3. 将执行传递到调用堆栈中的第一个catch块，如果没有，则终止程序执行
      • 错误对象
      • new Error()
      • new TypeError()
      • new ReferenceError()
      • ...
  • JS 的错误处理
    • 在同步情况下通过try..catch..来捕捉并处理错误对象
    • 在异步情况下，使用Promise.catch()和async+try进行错误处理
      • 因为异步函数执行时，try 语句的执行环境已经被释放，从而不能正常的进行错误处理
  • 通过继承Error基类来扩展自己的错误对象
const myError = new Error('oops');
myError.name; // Error
myError.message; // oops
myError.stack; // 堆栈信息

// 同步错误处理
try {
  throw new Error('fail');
} catch (e) {
  console.log(e);
}
// 异步错误处理
Promise.reject('fail')
  .catch(e => {
    console.log('mid: ', e);
  }) // mid: fail
  .catch(e => {
    console.log('fin: ', e);
  }); // 不会执行

// 扩展错误对象
class customError extends Error {
  constructor(message) {
    super(message);
    this.name = 'customError';
  }
}

const a = new customError('oops');
a.name; // customError
a.message; // oops
